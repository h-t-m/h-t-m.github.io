算法——初级排序

前言

零基础直接刷力扣还是有些虎了，所以特意补一下理论知识。本系列笔记将以 图灵社区 出版的 《算法：第4版》 为主要参考复现各种算法及实现过程，试图以自己的方式深度解析算法。

本文主要内容为选择排序、插入排序以及希尔排序。

需求

对于排序算法的需求很简单，就是排序就行了。作为验证，这里构造一个测试类 Demo ，包含判断序列是否按序排列的方法 isSorted() 。在 Java 中，几乎所有可用于排序的数据类型都会实现 Comparable 接口，而接口相当于一个绝对抽象的类，实现接口则相当于成为了 Comparable 的子类。也就是说，为保证通用性，对于 isSorted() 方法的参数，我们可以使用 Comparable 类型的序列：

对于 isSorted() 方法的实现，实际上只需要保证序列中的元素总是不小于前一位即可（默认升序）。而对于大小的比较，这里使用 Comparable 接口的 compareTo() 方法，该方法返回一个整数，正整数、零、负整数分别代表了大于、等于和小于，方法的具体实现由实现 Comparable 接口的类定义：

因此，后续排序算法只要通过 Demo 类测试，即可视为成功。（当然测试时要加 mian 方法）

选择排序

选择排序的思想非常简单，找出最小的元素放在并与第一个元素交换位置，然后在剩下的元素中找到最小的与第二个元素交换位置，当只剩下一个元素时，排序就完成了。可以使用两个 for 循环完成这个任务，内循环负责找出最小元素，外循环则负责将这个元素易位。这里使用 Selection 类封装选择排序，该类向外提供静态接口 sort 方法，该方法实现了选择排序：

选择排序算法的轨迹实例如下图：

选择排序的轨迹，图源自《算法：第4版》

虽说选择排序算法是最简单的，但是他的交换次数却比其他排序算法都要少。对于NNN 个元素的数组，最多只需进行NNN 次交换。此外，选择排序无论序列初始顺序如何，都必须进行(N−1)+(N−2)+...+2+1=N(N−1)/2(N-1)+(N-2)+...+2+1=N(N-1)/2(N−1)+(N−2)+...+2+1=N(N−1)/2 次比较。

插入排序

插入排序就是依次遍历数组，维护当前索引左侧的所有元素有序，而插入指的是当前元素会直接插入到左侧其正确的位置中去。在数组中，这种插入意味着要将全部大于当前元素的元素向右移动一位。

既然左侧始终保证有序，那么并不需要当前元素与左侧有序序列的每一位作比较。当前元素只需不断与前一位比较，若前一位更大则交换位置（前移），若前一位不大于当前元素则说明已到达正确位置，左侧有序达成！当右侧遍历完毕，数组排序也就完成了。

插入排序貌似与选择排序区别并不大，但是，当数组初始就有序时，选择排序依然需要N(N−1)/2N(N-1)/2N(N−1)/2 次比较，但插入排序却等价于执行一次 isSorted() 方法，对于两种算法更加详细的比较，可以参考 《算法：第4版》 。
